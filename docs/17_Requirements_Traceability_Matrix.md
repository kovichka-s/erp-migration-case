# 17. Requirements Traceability Matrix

## Мета документа

Requirements Traceability Matrix (RTM) використовується для забезпечення зв'язку між бізнес-потребами, вимогами, User Stories, Use Cases та критеріями прийняття.

Матриця дозволяє відстежувати, які бізнес-проблеми вирішуються кожною вимогою, та контролювати повноту реалізації функціональності в ERP-системі.

## Примітки

Requirements Traceability Matrix використовується як контрольний документ між бізнесом, аналітиком та командою розробки під час переходу від поточної ERP-системи до Odoo.

Матриця дозволяє:

* уникнути втрати вимог під час міграції;
* перевірити відповідність реалізації бізнес-потребам;
* забезпечити прозорість змін для стейкхолдерів.

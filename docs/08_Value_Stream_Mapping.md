# 08. Value Stream Mapping

## Purpose

Проаналізувати потік створення цінності (Value Stream) для процесу продажу пелет, визначити втрати (Waste), вузькі місця (Bottlenecks) та можливості покращення перед переходом на ERP Odoo.

---

# Scope

Від формування виробничого плану основної продукції
до відвантаження пелет клієнту.

---

# Relationship with BPMN

Даний документ побудований на основі BPMN AS IS (End-to-End).

BPMN використовується для розуміння загального виробничого процесу, тоді як Value Stream Mapping аналізує лише ті етапи, які впливають на швидкість продажу пелет.

Основний висновок BPMN:

> Обсяг виробництва пелет залежить від виробничого плану основної продукції (дошка, меблевий щит), оскільки саме він визначає кількість деревних відходів.

# Customer Value

Клієнт очікує:

- прогноз дати готовності партії;
- можливість планувати власні закупівлі;
- стабільність поставок;
- швидке підтвердження наявності продукції.

# Waste Analysis (Lean)

## Waiting

- очікування накопичення мінімальної партії;
- очікування інформації про готовність продукції.

## Inventory

- накопичення продукції на складі;
- ризик переповнення складу.

## Overproduction

- виробництво пелет швидше, ніж здійснюються продажі.

## Motion

- ручні дзвінки між керівником складу та менеджером з продажу.

## Defects

- брак пелет;
- пошкодження біг-бегів;
- некоректне фасування.

## Unused Human Potential

- менеджер з продажу витрачає час на ручне уточнення наявності продукції замість роботи з клієнтами.

---

# Bottleneck

Основне вузьке місце процесу — відсутність автоматичного прогнозування накопичення партії пелет.

Менеджер з продажу не може повідомити клієнту:

- дату готовності;
- доступний обсяг;
- прогноз наступної партії.

---

# Process Metrics

Для оцінки процесу необхідно відстежувати:

- Lead Time від початку виробництва до готової партії;
- Waiting Time до досягнення мінімального обсягу продажу;
- Кількість біг-бегів на складі;
- Темп виробництва пелет (біг-бегів/добу);
- Частоту переповнення складу;
- Частку відмов клієнтів через відсутність інформації про готовність.

---

# Improvement Opportunities

ERP повинна:

- прогнозувати дату накопичення мінімальної партії;
- прогнозувати обсяг виробництва пелет на основі виробничого плану;
- автоматично розраховувати доступний залишок;
- повідомляти менеджера з продажу про готовність партії;
- надавати прогнозовану дату відвантаження клієнту.

---

# Expected Business Value

Після впровадження рішення компанія повинна отримати:

- скорочення часу відповіді клієнту;
- зменшення ручної комунікації між підрозділами;
- зменшення ризику переповнення складу;
- прогнозованість продажів;
- покращення планування виробництва та логістики.

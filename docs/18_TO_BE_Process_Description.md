# 18. TO BE Process Description

## Мета документа

Документ описує цільовий процес (TO BE) після впровадження нової ERP-системи Odoo.

Основна мета змін — забезпечити прозоре управління процесом виробництва та продажу пелет, надати можливість прогнозувати доступність продукції та своєчасно інформувати клієнтів про можливі строки відвантаження.

---

# Опис цільового процесу

## 1. Планування доступності продукції

Після впровадження Odoo інформація про виробництво, складські залишки та готову продукцію об'єднується в єдиній системі.

Система дозволяє бачити:

* фактичний залишок готової продукції;
* продукцію у виробництві;
* прогнозований обсяг майбутнього виробництва;
* доступний обсяг для продажу;
* критичний рівень заповнення складу.

---

# 2. Формування доступного обсягу для продажу

Керівник складу пелет контролює:

* кількість готової продукції;
* якість продукції;
* наявність необхідного пакування;
* можливість формування економічно доцільного відвантаження.

Після досягнення мінімального обсягу для продажу система дозволяє сформувати інформацію про доступний товар.

Мінімальний обсяг відвантаження визначається бізнес-правилами компанії та враховує транспортні витрати.

---

# 3. Робота менеджера з продажу

Менеджер з продажу отримує актуальну інформацію про доступний товар.

На основі цих даних він:

* контактує з клієнтами;
* підтверджує можливість продажу;
* погоджує кількість продукції;
* формує замовлення клієнта.

Менеджер більше не залежить від ручного уточнення інформації у виробництва.

---

# 4. Формування замовлення клієнта

Після підтвердження замовлення:

1. Створюється замовлення клієнта в ERP.
2. Перевіряється наявність продукції.
3. Визначається спосіб доставки:

   * самовивіз клієнтом;
   * доставка транспортом компанії;
   * залучення зовнішнього перевізника.
4. Узгоджується вартість логістики.

---

# 5. Планування виробництва

Odoo враховує залежність виробництва пелет від основного виробничого процесу.

При прогнозуванні доступності продукції враховуються:

* тип основного виробництва;
* обсяг отриманих відходів;
* доступність сировини для виробництва пелет;
* виробнича потужність обладнання;
* залишки готової продукції.

Наприклад:

* виробництво дошки створює менший обсяг відходів;
* виробництво меблевого щита створює більший обсяг залишків, що збільшує потенційний вихід пелет.

---

# 6. Контроль пакувальних матеріалів

Система дозволяє контролювати:

* залишки мішків;
* залишки біг-бегів;
* потребу у закупівлі пакування.

Це дозволяє уникнути ситуацій, коли виробництво готове виготовляти продукцію, але процес зупиняється через відсутність пакування.

---

# 7. Відвантаження продукції

Після підтвердження замовлення:

* формується документ на відвантаження;
* склад готує продукцію;
* логіст організовує транспорт за необхідності;
* бухгалтерія контролює оплату та документи;
* продукція передається клієнту.

---

# Основні зміни після впровадження Odoo

| Поточний стан (AS IS)                             | Цільовий стан (TO BE)             |
| ------------------------------------------------- | --------------------------------- |
| Інформація зберігається у різних джерелах         | Єдина система управління даними   |
| Менеджер уточнює інформацію вручну                | Менеджер бачить актуальний статус |
| Немає точного прогнозу готовності продукції       | Є прогноз доступності продукції   |
| Ризик переповнення складу                         | Контроль критичних рівнів запасів |
| Відсутність зв'язку між виробництвом та продажами | Єдина інформаційна система        |
| Планування базується на досвіді працівників       | Планування базується на даних     |

---

# Очікуваний результат

Впровадження цільового процесу дозволяє:

* скоротити час відповіді клієнтам;
* підвищити точність планування продажів;
* зменшити ризик простою виробництва;
* уникнути переповнення складу;
* покращити взаємодію між виробництвом, продажами, логістикою та закупівлями.

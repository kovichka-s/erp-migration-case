# Functional Requirements

## Мета

Визначити функціональні вимоги до ERP-системи Odoo, які забезпечать підтримку процесу прогнозування готовності пелет, планування продажів та взаємодії між виробництвом, складом і відділом продажів.

---

# FR-001. Відображення поточного залишку готової продукції

## Опис

Система повинна відображати актуальну кількість готових пелет, доступних для продажу.

## Користувачі

- Менеджер з продажу
- Керівник складу пелет
- Керівник відділу продажів

## Критерії:

Система повинна показувати:

- кількість готових Big Bags;
- дату виготовлення;
- статус продукції (доступна / зарезервована / на перевірці).

---

# FR-002. Розрахунок прогнозу накопичення пелет

## Опис

Система повинна прогнозувати майбутній обсяг виробництва пелет на основі виробничих даних.

## Користувачі

- Керівник складу пелет
- Менеджер з продажу
- Керівник виробництва

## Критерії:

Прогноз повинен враховувати:

- план виробництва основної продукції;
- тип продукції;
- очікуваний обсяг відходів;
- поточний залишок сировини для виробництва пелет.

---

# FR-003. Врахування типу основної продукції

## Опис

Система повинна враховувати, що різні виробничі процеси створюють різний обсяг відходів.

## Користувачі

- Керівник виробництва
- Бізнес-аналітик
- Менеджер з продажу

## Критерії:

Для прогнозування повинні використовуватись дані:

- виробництво дошки;
- виробництво меблевого щита;
- інші процеси, які створюють деревні відходи.

---

# FR-004. Визначення доступної партії для продажу

## Опис

Система повинна автоматично визначати, коли кількість готової продукції достатня для формування пропозиції клієнту.

## Критерії:

Система повинна враховувати:

- мінімальний обсяг продажу;
- доступний складський залишок;
- резерви;
- продукцію в статусі перевірки якості.

---

# FR-005. Контроль місткості складу

## Опис

Система повинна контролювати поточне та прогнозоване заповнення складу пелет.

## Критерії:

Система повинна:

- показувати поточний рівень заповнення;
- розраховувати прогноз заповнення;
- створювати попередження при досягненні критичного рівня.

---

# FR-006. Повідомлення менеджера з продажу про готовність партії

## Опис

Система повинна повідомляти менеджера з продажу про можливість початку продажу.

## Критерії:

Повідомлення повинно містити:

- доступний обсяг;
- дату готовності;
- можливий обсяг відвантаження.

---

# FR-007. Планування відвантаження

## Опис

Система повинна підтримувати формування відвантаження відповідно до транспортних обмежень.

## Критерії:

Система повинна враховувати:

- кількість Big Bags;
- максимальну вантажопідйомність автомобіля;
- об'єм транспортного засобу.

---

# FR-008. Управління замовленнями клієнтів

## Опис

Система повинна дозволяти менеджеру з продажу створювати та супроводжувати замовлення клієнтів.

## Критерії:

Повинні зберігатися:

- клієнт;
- кількість продукції;
- дата запиту;
- статус виконання;
- планована дата відвантаження.

---

# FR-009. Інтеграція виробничих та складських даних

## Опис

Система повинна забезпечити єдине джерело інформації щодо руху продукції.

## Критерії:

Необхідно пов'язати:

виробництво → готову продукцію → склад → продаж → відвантаження.

---

# FR-010. Контроль наявності пакування

## Опис

Система повинна враховувати доступність пакувальних матеріалів перед плануванням виробництва та продажу.

## Критерії:

Повинна відображатися:

- кількість доступних Big Bags;
- потреба у закупівлі;
- прогноз дефіциту.

---

# FR-011. Формування аналітичних звітів

## Опис

Система повинна забезпечити аналітичне представлення ключових показників процесу.

## Користувачі

- Власник компанії
- Керівник відділу продажів
- Керівники виробничих напрямків

## Звіти:

- залишки готової продукції;
- прогноз виробництва;
- швидкість накопичення пелет;
- завантаження складу;
- виконання продажів.

---

# FR-012. Історія змін даних

## Опис

Система повинна зберігати історію змін ключових параметрів процесу.

## Критерії:

Повинна бути доступна інформація:

- хто змінив дані;
- коли внесено зміни;
- які значення були змінені.

---

# Висновок

Функціональні вимоги спрямовані на створення єдиного інформаційного середовища, яке дозволить перейти від ручного управління процесом продажу пелет до прогнозованого управління на основі актуальних виробничих даних.

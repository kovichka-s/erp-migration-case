# 23. Risk Analysis and Assumptions

## Мета документа

Документ визначає ризики, припущення (Assumptions), обмеження (Constraints) та залежності (Dependencies), які можуть вплинути на успішність впровадження функціоналу в ERP Odoo.

---

# Assumptions (Припущення)

| ID   | Припущення                                                         | Вплив у разі невиконання                            |
| ---- | ------------------------------------------------------------------ | --------------------------------------------------- |
| A-01 | ERP Odoo буде впроваджена для всіх підрозділів компанії            | Дані залишаться розподіленими між різними системами |
| A-02 | Менеджер з продажу своєчасно вносить замовлення до системи         | Прогноз виробництва буде неточним                   |
| A-03 | Керівник складу підтримує актуальність залишків продукції          | Дані ERP втратять достовірність                     |
| A-04 | Виробництво регулярно передає інформацію про випуск продукції      | Неможливо прогнозувати доступний обсяг              |
| A-05 | Закупівлі забезпечують своєчасне постачання пакувальних матеріалів | Можлива зупинка виробництва пелет                   |
| A-06 | Користувачі пройдуть навчання перед початком роботи в новій ERP    | Зростання кількості помилок та опору змінам         |

---

# Constraints (Обмеження)

| ID   | Обмеження                                                                                 | Вплив                                                        |
| ---- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
| C-01 | Проєкт реалізується в межах стандартної функціональності Odoo з мінімальною кастомізацією | Частину побажань бізнесу може бути відкладено                |
| C-02 | Виробництво пелет залежить від основного виробництва                                      | Неможливо виготовити продукцію незалежно від обсягу відходів |
| C-03 | Максимальний обсяг відвантаження визначається транспортними нормами                       | Неможливо перевищити допустиме навантаження автомобіля       |
| C-04 | Мінімально економічно доцільне відвантаження становить 10 біг-бегів                       | Дрібні замовлення можуть бути економічно невигідними         |
| C-05 | Склад має обмежену місткість                                                              | Неможливо накопичувати необмежений запас готової продукції   |

# Основні ризики проєкту

Найбільший вплив на успішність рішення мають:

* якість даних у ERP;
* своєчасне оновлення інформації відповідальними особами;
* безперервне забезпечення пакувальними матеріалами;
* синхронізація роботи виробництва, складу та продажів;
* готовність користувачів працювати за новими процесами.

---

# Висновок

Більшість виявлених ризиків пов'язані не з технічною реалізацією ERP, а з організацією бізнес-процесів та якістю даних.

Передбачені припущення, обмеження та заходи реагування дозволяють знизити вплив цих ризиків і підвищити ймовірність успішного впровадження рішення.

# 20. Gap Analysis

## Мета документа

Gap Analysis використовується для визначення різниці між поточним станом процесу (AS IS) та цільовим станом після впровадження ERP-системи Odoo (TO BE).

Основна мета аналізу — визначити, які зміни необхідні для усунення існуючих проблем, підвищення прозорості процесів та покращення управління виробництвом і продажами пелет.

# Основні категорії розривів

## 1. Data Gap (розрив даних)

Проблема:

* інформація знаходиться у різних джерелах;
* відсутня єдина актуальна картина процесу.

Рішення:

* централізація даних в ERP;
* єдина база для виробництва, складу та продажів.

---

## 2. Process Gap (розрив процесів)

Проблема:

* багато ручних операцій;
* залежність від досвіду конкретних працівників;
* відсутність формалізованих правил.

Рішення:

* опис процесів;
* налаштування стандартних сценаріїв роботи;
* автоматизація передачі інформації.

---

## 3. Visibility Gap (розрив прозорості)

Проблема:

Менеджер з продажу не має можливості швидко відповісти клієнту:

* коли буде доступний необхідний обсяг;
* чи можливо сформувати відвантаження;
* який прогноз виробництва.

Рішення:

* створення прозорого статусу продукції;
* доступ до актуальної інформації для відповідальних ролей.

---

## 4. Planning Gap (розрив планування)

Проблема:

Виробництво пелет залежить від основного виробничого процесу, але ця залежність не використовується для прогнозування.

Рішення:

* врахування обсягів виробництва основної продукції;
* прогнозування утворення відходів;
* планування майбутньої доступності пелет.

---

# Очікуваний результат після усунення Gap

Після впровадження змін компанія отримує:

* прогнозований процес продажу пелет;
* можливість своєчасно інформувати клієнтів;
* зменшення ризику втрати замовлень;
* контроль виробничих та складських обмежень;
* прозору взаємодію між підрозділами;
* основу для подальшої автоматизації та аналітики.

---

# Висновок

Основний розрив полягає не у відсутності виробничих можливостей, а у відсутності єдиного інформаційного простору між виробництвом, складом, продажами, закупівлями та логістикою.

Впровадження Odoo дозволяє перейти від реактивного управління процесом до прогнозованого управління на основі даних.

# 25. Open Questions

## Мета документа

Open Questions використовується для фіксації відкритих питань, на які ще не отримано остаточну відповідь від бізнесу. Документ допомагає контролювати незавершені рішення, планувати додаткові інтерв'ю зі стейкхолдерами та уникати неоднозначностей під час розробки і впровадження ERP-системи.

# Правила ведення документа

* Кожне питання має унікальний ідентифікатор.
* Після отримання відповіді питання переноситься до відповідного артефакту (Business Rules, Functional Requirements, User Stories, Decision Log тощо).
* Після перенесення статус змінюється на **Closed**, але запис залишається в журналі для історії.
* Open Questions регулярно переглядається під час сесій збору вимог.

---

# Висновок

Open Questions допомагає системно керувати невизначеностями, не втрачати важливі бізнес-питання та забезпечує повну простежуваність процесу уточнення вимог протягом усього життєвого циклу проєкту.

# 14. User Stories

## Мета документа

Цей документ описує потреби користувачів системи у форматі User Stories для майбутньої ERP-системи. User Stories сформовані на основі виявленої бізнес-проблеми: необхідності прогнозування доступності продукції та своєчасного планування продажів і відвантажень пелет.

---

## User Story 1 — Прогноз доступності продукції для продажу

**Як** менеджер з продажу
**Я хочу** бачити прогноз доступної кількості пелет та очікувані строки готовності наступної партії
**Щоб** надавати клієнтам достовірну інформацію щодо можливості відвантаження.

### Acceptance Criteria

* Система відображає поточну кількість готової продукції на складі.
* Система показує продукцію, яка знаходиться у виробництві.
* Користувач може побачити прогнозований строк досягнення мінімального об'єму відвантаження.
* Інформація доступна без додаткового збору даних вручну з різних джерел.

---

## User Story 2 — Контроль наповнення складу готової продукції

**Як** керівник складу пелет
**Я хочу** контролювати поточний обсяг готової продукції та прогноз заповнення складу
**Щоб** своєчасно передавати інформацію у відділ продажу та уникати переповнення складу.

### Acceptance Criteria

* Система показує поточну кількість готової продукції.
* Система дозволяє визначити критичний рівень заповнення складу.
* При наближенні до критичного рівня користувач отримує інформацію для планування продажу.
* Враховується мінімальний економічно доцільний обсяг відвантаження.

---

## User Story 3 — Планування машинного відвантаження

**Як** менеджер з продажу
**Я хочу** знати доступний обсяг продукції для формування машинного відвантаження
**Щоб** пропонувати клієнтам оптимальний обсяг замовлення з урахуванням транспортних обмежень.

### Acceptance Criteria

* Система дозволяє визначити доступний обсяг для завантаження автомобіля.
* Враховуються обмеження вантажопідйомності транспорту.
* Користувач може сформувати замовлення відповідно до оптимального обсягу перевезення.
* Дані можуть використовуватися для попереднього планування логістики.

---

## User Story 4 — Врахування виробничого прогнозу

**Як** керівник виробництва пелет
**Я хочу** бачити залежність обсягу майбутнього виробництва від основного виробничого процесу
**Щоб** прогнозувати майбутню доступність сировини для виготовлення пелет.

### Acceptance Criteria

* Система враховує джерело утворення відходів основного виробництва.
* Враховується тип основної продукції, яка виготовляється (дошка, меблевий щит тощо).
* Можна оцінити потенційний обсяг утворення відходів для виробництва пелет.
* Прогноз може використовуватися для планування виробництва.

---

## User Story 5 — Контроль забезпечення виробництва пакувальними матеріалами

**Як** керівник виробництва пелет
**Я хочу** бачити доступну кількість пакувальних матеріалів
**Щоб** уникнути простою виробництва через відсутність мішків або біг-бегів.

### Acceptance Criteria

* Система відображає залишки пакувальних матеріалів.
* Визначається мінімальний необхідний запас.
* Користувач може бачити потребу у закупівлі.
* Інформація доступна менеджеру закупівель.

---

## User Story 6 — Планування відвантаження клієнту

**Як** менеджер з продажу
**Я хочу** створювати замовлення на основі фактичної готовності продукції
**Щоб** уникати обіцянок клієнту без підтвердження можливості виконання.

### Acceptance Criteria

* Замовлення створюється тільки після підтвердження доступності продукції.
* Користувач бачить статус готовності замовлення.
* Інформація про необхідність логістики передається відповідальним особам.
* Статус замовлення можна відстежувати до моменту відвантаження.

---

## User Story 7 — Аналіз ефективності процесу

**Як** власник компанії
**Я хочу** бачити ключові показники процесу виробництва та продажу
**Щоб** приймати управлінські рішення на основі актуальних даних.

### Acceptance Criteria

* Доступні показники виробництва, складу та продажів.
* Дані можуть використовуватися для аналізу відхилень.
* Можна оцінити вплив покращень процесу на бізнес-результати.

---

## Примітки

User Stories є основою для подальшого формування:

* Use Cases;
* Acceptance Criteria;
* Requirements Traceability Matrix;
* TO BE процесу в ERP Odoo.

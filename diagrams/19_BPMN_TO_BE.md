# 19. BPMN TO BE

## Мета документа

Цей документ містить модель **TO BE** бізнес-процесу, який буде реалізовано після впровадження ERP-системи Odoo.

На відміну від **BPMN AS IS**, який описує повний наскрізний (End-to-End) виробничий процес підприємства, дана діаграма відображає **лише процес, що змінюється в межах поточного Scope проєкту**.

---

# Scope моделювання

До моделі **TO BE** входять лише зміни, пов'язані з процесом продажу пелет та прогнозуванням готовності продукції.

Основні покращення:

* менеджер з продажу отримує інформацію про залишки продукції без додаткових уточнень;
* ERP розраховує прогноз доступності продукції;
* менеджер може повідомити клієнту орієнтовну дату готовності замовлення;
* враховуються виробничі залишки та доступність пакувальних матеріалів;
* зменшується кількість ручних комунікацій між підрозділами.

# Відмінності від AS IS

| AS IS                                                | TO BE                                     |
| ---------------------------------------------------- | ----------------------------------------- |
| Менеджер телефонує на склад для уточнення інформації | Дані доступні в ERP                       |
| Відсутній прогноз готовності продукції               | ERP автоматично формує прогноз            |
| Інформація збирається вручну з кількох джерел        | Вся інформація доступна в одному місці    |
| Високий ризик помилки через людський фактор          | Мінімізація ручних операцій               |
| Частина інформації передається усно                  | Дані зберігаються та відображаються в ERP |

# Висновок

Діаграма **BPMN TO BE** демонструє цільовий бізнес-процес після впровадження Odoo в межах поточного Scope. Вона фокусується виключно на процесах, які змінюються, та не дублює повну End-to-End модель виробництва, представлену в BPMN AS IS.
